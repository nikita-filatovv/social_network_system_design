# System Design социальной сети для курса по System Design

___
## Функциональные требования
1. Публикация постов с вложениями
    - Фотографии
    - Текстовое описание (небольшое)
    - Привязка к локации
2. Взаимодействие пользователей с постами других пользователей
    - Оценка постов
    - Комментарии к постам
3. Система подписки одних юзеров на других
    - Действие подписки
    - Действие отписки
    - (следить за активностью)
4. Поиск популярных мест для путешествий
    - Определение популярности мест
    - Поиск популярных мест
    - Просмотр постов из этих мест
5. Просмотр ленты, основанной на подписках (в обратном хронологическом порядке)
    - Просмотр своей ленты
    - Просмотр ленты других пользоватей

## Нефункциональные требования
1. DAU - 10000000
    - Линейный рост до этого показателя на дистанции года
    - Через год планируется дальнейший рост (масштабируема)
    - Примерные показатели, взятые для расчета:
        1. 4 поста от пользователя в год - 0,02 (грубо в день)
        2. 10 постов в день читает пользователь, ставит реакцию и коммент
2. География - страны СНГ
    - Без плана выхода на глобальный рынок
3. Платформы
    - МП
    - Браузер
4. RPS (примерно)
    - Посты (write) - 2,5
    - Посты (read) - 1150
    - Реакции (post) - 1150
    - Реакции (read) - 1150
    - Комментарии (post) - 1150
    - Комментарии (read) -  23000 
5. Traffic (примерно)
    - Посты (write) -  50кб * 2,5 = 125кб/с
    - Посты (read) - 65кб * 1150 = 73мб/с
    - Реакции (post) - 0.2кб * 1150 = 230кб/с
    - Реакции (read) - 0.2кб * 1150 = 230кб/с
    - Комментарии (post) - 3кб * 1150 = 3.4мб/с
    - Комментарии (read) - 3кб * 23000 = 67мб/с
6. Доступность системы - 99,9%
